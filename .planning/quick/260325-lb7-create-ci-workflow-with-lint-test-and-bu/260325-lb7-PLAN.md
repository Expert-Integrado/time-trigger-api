Criar o arquivo .github/workflows/ci.yml com o seguinte conteudo (nao modificar docker-publish.yml):

```yaml
name: CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          run_install: false

      - name: Get pnpm store directory
        id: pnpm-cache
        run: echo "STORE_PATH=$(pnpm store path --silent)" >> $GITHUB_OUTPUT

      - name: Cache pnpm store
        uses: actions/cache@v4
        with:
          path: ${{ steps.pnpm-cache.outputs.STORE_PATH }}
          key: ${{ runner.os }}-pnpm-store-${{ hashFiles('**/pnpm-lock.yaml') }}
          restore-keys: |
            ${{ runner.os }}-pnpm-store-

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm run lint

      - name: Test
        run: pnpm run test

      - name: Build
        run: pnpm run build
```

Observacoes de implementacao:
- `pnpm/action-setup@v4` detecta a versao do pnpm via campo `packageManager` no package.json (se existir) ou usa a versao mais recente estavel
- Cache do pnpm store evita re-download em execucoes consecutivas
- `--frozen-lockfile` garante reproducibilidade (falha se pnpm-lock.yaml estiver desatualizado)
- Os steps de lint e test podem falhar o job automaticamente se retornarem exit code != 0
- O step lint usa `pnpm run lint` que internamente chama `eslint ... --fix` — no CI isso pode gerar diff sem commit; considerar remover --fix do script de lint para CI ou aceitar o comportamento (lint falha apenas se houver erros nao-auto-fixaveis)

**1. Create `src/dispatch/message-check.service.ts`** (new file, per D-07, D-08):

```typescript
import { Injectable } from '@nestjs/common';
import { Db } from 'mongodb';

@Injectable()
export class MessageCheckService {
  async hasProcessingMessage(
    db: Db,
    botIdentifier: string,
    chatDataId: string,
  ): Promise<boolean> {
    const doc = await db.collection('messages').findOne({
      botIdentifier,
      chatDataId,
      messageStatus: 'processing',
    });
    return doc !== null;
  }
}
```

Key points:
- Uses `findOne` (not `countDocuments`) — stops at first match (per research anti-pattern guidance)
- Query always includes all three fields: `botIdentifier`, `chatDataId`, `messageStatus: 'processing'` (DEP-04, DEP-05, D-02, D-03)
- Stateless — accepts `Db` as parameter, no internal state

Position rationale: rate limit is in-memory (free), dependency check is a DB query (has cost). Rate limit first avoids unnecessary MongoDB queries.

Insert this block after the rate limit `break` and before `const claimed = await this.webhookDispatchService.dispatch(...)`:

```typescript
// DEP-02, DEP-03, DEP-04, DEP-05: dependency guard — block run if matching message is processing
const botIdentifier = run['botIdentifier'] as string | undefined;
const chatDataId = run['chatDataId'] as string | undefined;
if (botIdentifier && chatDataId) {
  const blocked = await this.messageCheckService.hasProcessingMessage(
    db,
    botIdentifier,
    chatDataId,
  );
  if (blocked) {
    this.logger.warn(
      `[${dbName}] Run ${String(run['_id'])} blocked — message still processing (chatDataId: ${chatDataId}, botIdentifier: ${botIdentifier})`,
    );
    continue;
  }
}
```

Key points per locked decisions:
- `botIdentifier` comes from `run['botIdentifier']` directly, NOT from vars (D-01)
- Query uses both fields always (D-02, DEP-04)
- Only `'processing'` blocks, not `'pending'` (D-03, DEP-05)
- Blocked run stays `"waiting"` — no status mutation, `continue` skips to next run (DEP-03)
- Log level is `warn` (D-11) with the exact format from D-10
- If either `botIdentifier` or `chatDataId` is absent on the run document, skip the check silently and proceed to dispatch (safe default for legacy documents — Claude's discretion)

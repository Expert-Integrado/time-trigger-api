<objective>
Split `RunDispatchService.runCycle()` into 3 independent public methods — `runRunsCycle()`, `runFupCycle()`, and `runMessagesCycle()` — each with its own `isRunning` guard so they can never block each other.

Purpose: Today a slow runs cycle prevents FUP and messages from being dispatched. With 3 independent guards, each dispatch type runs on its own rhythm without cross-blocking.
Output: Updated `run-dispatch.service.ts` with 3 public cycle methods + updated spec with tests for independence.
</objective>

   - Guard: `if (this.isRunningRuns)` → warn 'Runs cycle skipped — previous cycle still running' and return
   - Set `this.isRunningRuns = true` in try, reset in finally

4. Add `public async runFupCycle(): Promise<void>`:
   - Guard: `if (this.isRunningFup)` → warn 'FUP cycle skipped — previous cycle still running' and return
   - Set `this.isRunningFup = true` in try, reset in finally
   - Logic: get eligible DBs → `Promise.allSettled` over `this.processDatabaseFup(dbName)`
   - Log: 'FUP cycle started' / 'FUP cycle complete — N DBs, N errors'

5. Add `public async runMessagesCycle(): Promise<void>`:
   - Guard: `if (this.isRunningMessages)` → warn 'Messages cycle skipped — previous cycle still running' and return
   - Set `this.isRunningMessages = true` in try, reset in finally
   - Logic: get eligible DBs → `Promise.allSettled` over `this.processDatabaseMessages(dbName)`
   - Log: 'Messages cycle started' / 'Messages cycle complete — N DBs, N errors'

6. Split current `processDatabase()` into 3 private methods:

   `private async processDatabaseRuns(dbName: string): Promise<void>`:
   - Read vars → check timeTrigger (enabled, time window, allowed days)
   - Read webhooks → get 'Processador de Runs' URL
   - Query runs collection: `{ runStatus: 'waiting', waitUntil: { $lte: Date.now() } }`
   - For each run: `this.webhookDispatchService.dispatch(db, run, webhookUrl)`
   - ALSO dispatch FUPs inside this same method (same timeTrigger gate scope):
     - Get 'Gerenciador follow up' URL from same webhookDoc
     - Query fup: `{ status: 'on', nextInteractionTimestamp: { $lte: Date.now() } }`
     - For each fup: `this.webhookDispatchService.dispatchFup(db, fup, fupUrl)`
   - NOTE: runRunsCycle processes BOTH runs AND fups (they share the timeTrigger gate)

   `private async processDatabaseFup(dbName: string): Promise<void>`:
   - Read vars → check timeTrigger (same gates as runs)
   - Read webhooks → get 'Gerenciador follow up' URL
   - Query fup: `{ status: 'on', nextInteractionTimestamp: { $lte: Date.now() } }`
   - For each fup: `this.webhookDispatchService.dispatchFup(db, fup, fupUrl)`
   - This is a standalone FUP-only method for when the FUP interval fires

   `private async processDatabaseMessages(dbName: string): Promise<void>`:
   - Read webhooks → get 'mensagens pendentes' URL (NO vars/timeTrigger check)
   - Query messages: `{ messageStatus: 'pending' }`
   - For each message: `this.webhookDispatchService.dispatchMessage(db, message, url)`

7. Delete old `runCycle()` and old `processDatabase()` entirely.

8. Keep `isWithinTimeWindow()` and `isAllowedDay()` private helpers (used by all 3 processDatabase* methods that check timeTrigger).

**Important conventions:**

- Single quotes, trailing commas
- `private readonly logger = new Logger(RunDispatchService.name)`
- NestJS Logger — use `this.logger.log()`, `this.logger.warn()`, `this.logger.error()`

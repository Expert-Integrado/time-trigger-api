- Change return type to `Promise<boolean>`
- After successful post + findOneAndUpdate: if `result` is truthy (document found), `return true`; if `result` is null (already claimed), log warning as before and `return false`
- After failed post (retry path): `return false` — the retry fires async via setTimeout and is outside the cycle scope per D-03
- The retry logic (setTimeout fire-and-forget) stays unchanged — it continues returning void internally

- Same pattern: `Promise<boolean>`, return true only when findOneAndUpdate returns a document, false otherwise

For `dispatchMessage(db, message, webhookUrl)`:
- Same pattern: `Promise<boolean>`, return true only when findOneAndUpdate returns a document, false otherwise

Key: The boolean reflects the atomic claim result, NOT the HTTP post result. A successful post where findOneAndUpdate returns null (already claimed) returns false.

**In src/dispatch/run-dispatch.service.ts:**

1. Add rate limit properties to the class. Read limits from process.env with default 10 per D-07:
```typescript
private readonly rateLimitRuns = parseInt(process.env['RATE_LIMIT_RUNS'] ?? '10', 10);
private readonly rateLimitFup = parseInt(process.env['RATE_LIMIT_FUP'] ?? '10', 10);
private readonly rateLimitMessages = parseInt(process.env['RATE_LIMIT_MESSAGES'] ?? '10', 10);
```

2. In `processDatabaseRuns(dbName)`:
- Add `let counterRuns = 0;` and `let counterFup = 0;` at the top of the method (per D-04, D-05 — local variables = per-database per-cycle scope, satisfying RATE-01 and RATE-07)
- In the `for (const run of runs)` loop:
  - Before dispatching, check: `if (counterRuns >= this.rateLimitRuns) { this.logger.warn(\`[\${dbName}] Rate limit reached for runs (\${counterRuns}/\${this.rateLimitRuns}) — skipping remaining items\`); break; }` (per D-06, D-11)
  - Change `await this.webhookDispatchService.dispatch(db, run, webhookUrl);` to capture return: `const claimed = await this.webhookDispatchService.dispatch(db, run, webhookUrl);`
  - If `claimed` is true, increment: `counterRuns++;` (per D-02, RATE-06)
- In the `for (const fup of fups)` loop:
  - Before dispatching, check: `if (counterFup >= this.rateLimitFup) { this.logger.warn(\`[\${dbName}] Rate limit reached for FUP (\${counterFup}/\${this.rateLimitFup}) — skipping remaining items\`); break; }`
  - Capture return: `const claimed = await this.webhookDispatchService.dispatchFup(db, fup, fupWebhookUrl);`
  - If `claimed` is true: `counterFup++;`
- After both loops (at end of method), log per D-10:
  ```
  this.logger.log(`[${dbName}] Runs: ${counterRuns}/${this.rateLimitRuns} dispatched`);
  this.logger.log(`[${dbName}] FUP: ${counterFup}/${this.rateLimitFup} dispatched`);
  ```

3. In `processDatabaseFup(dbName)`:
- Add `let counterFup = 0;` at the top
- In the `for (const fup of fups)` loop:
  - Check limit before dispatch: `if (counterFup >= this.rateLimitFup) { this.logger.warn(\`[\${dbName}] Rate limit reached for FUP (\${counterFup}/\${this.rateLimitFup}) — skipping remaining items\`); break; }`
  - Capture: `const claimed = await this.webhookDispatchService.dispatchFup(db, fup, fupWebhookUrl);`
  - If claimed: `counterFup++;`
- After loop, log: `this.logger.log(\`[\${dbName}] FUP: \${counterFup}/\${this.rateLimitFup} dispatched\`);`

4. In `processDatabaseMessages(dbName)`:
- Add `let counterMessages = 0;` at the top
- In the `for (const message of messages)` loop:
  - Check limit: `if (counterMessages >= this.rateLimitMessages) { this.logger.warn(\`[\${dbName}] Rate limit reached for messages (\${counterMessages}/\${this.rateLimitMessages}) — skipping remaining items\`); break; }`
  - Capture: `const claimed = await this.webhookDispatchService.dispatchMessage(db, message, messagesWebhookUrl);`
  - If claimed: `counterMessages++;`
- After loop, log: `this.logger.log(\`[\${dbName}] Messages: \${counterMessages}/\${this.rateLimitMessages} dispatched\`);`

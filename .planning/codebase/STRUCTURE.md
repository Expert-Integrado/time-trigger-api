## Directory Purposes

**src/:**
- Purpose: All application source code
- Contains: Controllers, services, modules, business logic
- Key files: `main.ts` (entry point), `app.module.ts` (module definition)

**test/:**
- Purpose: End-to-end test suite
- Contains: E2E test files and E2E-specific Jest configuration
- Key files: `app.e2e-spec.ts` (HTTP integration tests)

**dist/:**
- Purpose: Compiled JavaScript output (do not commit)
- Contains: JavaScript and source maps from TypeScript compilation
- Generated by: `npm run build` command

**coverage/:**
- Purpose: Test coverage reports (do not commit)
- Contains: Coverage metrics and HTML reports
- Generated by: `npm run test:cov` command

## Key File Locations

**Entry Points:**
- `src/main.ts`: Application bootstrap - creates NestJS app and starts HTTP server on PORT env var (default 3000)

**Configuration:**
- `tsconfig.json`: TypeScript compiler options (ES2023 target, strict null checks enabled)
- `nest-cli.json`: NestJS CLI settings (source root is `src/`)
- `package.json`: Project dependencies and npm scripts

**Core Logic:**
- `src/app.module.ts`: Root module defining controllers and providers
- `src/app.controller.ts`: HTTP route handlers, single GET / endpoint
- `src/app.service.ts`: Business logic service with getHello() method

**Testing:**
- `src/app.controller.spec.ts`: Unit test for AppController using Jest
- `test/app.e2e-spec.ts`: End-to-end HTTP test using Supertest
- `test/jest-e2e.json`: Jest configuration for E2E tests

## Naming Conventions

**Files:**
- Controllers: `*.controller.ts` (e.g., `app.controller.ts`)
- Services: `*.service.ts` (e.g., `app.service.ts`)
- Modules: `*.module.ts` (e.g., `app.module.ts`)
- Unit tests: `*.spec.ts` (e.g., `app.controller.spec.ts`)
- E2E tests: `*.e2e-spec.ts` (e.g., `app.e2e-spec.ts`)
- Main file: `main.ts`

**Directories:**
- Feature modules: Named after feature (e.g., `users/`, `posts/`)
- Test directory: `test/` for E2E tests
- Source directory: `src/`

## Where to Add New Code

**New Feature:**
- Create feature directory: `src/features/[feature-name]/`
- Primary code: `src/features/[feature-name]/[feature-name].controller.ts` (routes) and `[feature-name].service.ts` (logic)
- Tests: `src/features/[feature-name]/[feature-name].controller.spec.ts`
- Module: `src/features/[feature-name]/[feature-name].module.ts`
- Import module in `src/app.module.ts` under `imports: []`

**New Component/Module:**
- Create new `*.controller.ts`, `*.service.ts`, `*.module.ts` files in feature directory
- Decorate controller with `@Controller('route')` for custom path
- Decorate service with `@Injectable()`
- Register in module's `controllers: []` and `providers: []` arrays

**Utilities:**
- Shared helpers: `src/common/` or `src/utils/`
- Shared guards: `src/guards/`
- Shared pipes: `src/pipes/`
- Shared decorators: `src/decorators/`

## Special Directories

**coverage/:**

**node_modules/:**

- Committed: No (added to .gitignore)
